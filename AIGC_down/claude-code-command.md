# AIGC_down — 论文降重与AIGC痕迹修订

根据用户提供的查重报告和AIGC检测报告，修改 thesis-writer 生成的 Markdown 论文，提升原创性和自然学术表达。

$ARGUMENTS

## 工作流程

### 1. 定位材料

1. 检查当前项目中的 `查重/` 文件夹（或用户指定的报告路径）。
2. 识别报告类型：
   - 查重报告：重复/高重叠段落、来源、标红文本、百分比或匹配片段
   - AIGC报告：AI标记段落、标红文本、风险评分或可疑段落
3. 定位 thesis-writer 生成的 `.md` 论文文件。
4. 读取足够的论文上下文，避免改变含义或破坏论证。

报告可以是 `.html`、`.pdf`、`.docx`、`.md`、`.txt` 或导出的文本。

### 2. 构建修订映射

为需要修订的段落创建工作映射：
- 来源类型：查重报告、AIGC报告或两者
- 原始论文段落
- 所在章节标题
- 修订原因
- 约束条件：引用、公式、变量、专有名词或术语必须保持不变

### 3. 先处理高重复段落

对每个高重复段落：
1. 按自然段落拆分，逐段处理。
2. 每个改写单元不超过500字；超过则按句子边界拆分。
3. 提取当前段落的关键点、逻辑和结论。
4. 压缩为简短内部大纲。
5. 从大纲重写为新段落，长度大致相近。
6. 调整句子顺序、解释路径和连接结构。
7. 保持专业术语、公式、引用、名称和事实声明的准确性。
8. 在观点明显依赖来源时添加或保留引用。

### 4. 再处理AIGC标记段落

如果用户提供了自定义润色提示词（$ARGUMENTS），使用它作为风格指导（不得与学术诚信冲突）。否则使用以下默认指导：

```
请对这段内容进行学术化润色，使其更贴近本科论文中自然、平实、通顺的表达方式。保留原意、专业术语、数据、引用和逻辑关系，避免过度模板化和生硬堆砌。可以调整句子前后结构，适当使用长短句交替，把过于标准化或重复的表达改成更自然的同义表达；专业词密集的句子可以适当拆短，说明性句子可以适当展开。不要使用"我""我们"等第一人称，不要编造新结论、数据或来源。
```

处理规则：
- 逐段处理，不要一次性改写所有标记段落。
- 保持本科论文的正式学术语气。
- 避免添加无意义的填充内容。
- 保持与前后段落的章节连贯性。

### 5. 同时被两份报告标记的段落

1. 先应用查重重写（关键点重构）。
2. 再对结果进行自然学术风格润色。
3. 重新检查术语、引用和含义是否保留。

### 6. 生成修订文件

在原论文同目录生成新文件，文件名示例：
- `<原名>_AIGC_down.md`
- `<原名>_降重润色版.md`

**不要覆盖原文件**，除非用户明确要求。

### 7. 输出修订摘要

报告以下内容：
- 原论文文件路径
- 新 Markdown 文件路径
- 使用的报告
- 修改的章节列表
- 需要人工审查的段落（引用、公式或来源上下文不足的声明）

## 安全与诚信边界

- 不承诺新文件一定能通过查重或AIGC检测。
- 不帮助隐藏未标注引用的复制文本。
- 如段落明显复制自某来源，优先改写+引用，或建议添加引用。
- 保留事实、公式、术语、引用、图表引用、章节编号和 Markdown 结构。
- 不编造来源、数据、实验结果或结论。
- 不为使文本看起来不同而删除必要引用。

## 修订阈值

- 查重率 >= 10%：优先处理所有高重叠和标红段落。
- 查重率 < 10%：仍修订报告中明确列出或标红的段落，但范围保守。
- AIGC指标 >= 10%：优先处理所有AI标记/高风险段落。
- AIGC指标 < 10%：仍修订明确标记的段落，但改动轻微。
- 两项均 < 10% 且无具体段落可识别：询问用户是否进行轻度全文润色。

## 质量检查清单

- [ ] 修订后的 `.md` 文件在原论文同目录
- [ ] Markdown 标题、列表、表格、代码块、公式、引用、图表引用和编号有效
- [ ] 重复段落从关键点重写，而非仅替换同义词
- [ ] AIGC标记段落改为自然学术表达，无无意义填充
- [ ] 未编造新事实、来源、数据或结果
- [ ] 原文件保持不变（除非明确授权）
